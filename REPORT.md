# Отчёт по проекту: классификация тарелок PlatesV2

**Соревнование:** [Cleaned vs Dirty V2](https://www.kaggle.com/c/platesv2)
**Автор:** Николай Видякин
**Финальный результат:** public LB accuracy = **0.83467**

---

## 1. Описание задачи и постановка целей

**Задача.** Бинарная классификация изображений тарелок на два класса: `cleaned` (чистая) и `dirty` (грязная).

**Данные.**

| | Train | Test |
|---|---|---|
| Количество изображений | 40 | 744 |
| Соотношение | 1 | 18.6× |
| Разметка | cleaned (20) / dirty (20) | нет (закрытый тест) |
| Формат | JPG, 256–455 px | JPG, аналогично |

**Главная сложность.** Тренировочных данных в **18 раз меньше**, чем тестовых. Это few-shot задача: обучить CNN с нуля невозможно — любая сеть запомнит 40 картинок наизусть и провалит тест. Требуется **transfer learning + сильная регуляризация + кросс-валидация**.

**Метрика.** Accuracy на скрытом тесте (kaggle public/private leaderboard).

**Цель.** Получить устойчивый baseline accuracy ≥ 0.80 на public LB с корректной методологией оценки и описанием ограничений и путей улучшения.

---

## 2. Обоснование подхода

### 2.1. Инструменты

- **Python 3.12, PyTorch 2.6 + CUDA 12.4** — стандартный DL-стек, предобученные модели из `torchvision`.
- **scikit-learn** — `StratifiedKFold`, метрики (accuracy, F1, confusion matrix).
- **matplotlib / PIL** — визуализация и работа с изображениями.
- **Аппаратное обеспечение:** NVIDIA RTX 4050 Laptop (6 GB). Полный 5-fold прогон — ~4 минуты.

### 2.2. Архитектура — ResNet18 + frozen backbone

Финальная модель: **ResNet18**, предобученный на ImageNet, с полностью замороженным backbone. Заменён только последний полносвязный слой:

```
ResNet18 backbone (frozen):                 ~11.69M params, requires_grad=False
└── fc:  Dropout(0.3) → Linear(512 → 2):    1026 trainable params
```

**Сравнение проверенных вариантов** (все — 5-fold + TTA, public LB на kaggle):

| Вариант | Public LB | Комментарий |
|---|---|---|
| ResNet18, frozen backbone | **0.8347** | финальный выбор: воспроизводимо |
| ResNet18, разморожен layer4 | 0.7245 | переобучение: train acc → 1.0, val застревает |
| ResNet50, frozen + разморожен layer4 (один прогон) | 0.8535 | разброс по фолдам std=0.146 — выше, чем у ResNet18 (0.127); результат частично seed-зависим |
| Ансамбль ResNet18 + ResNet50 | 0.7298 | нестабильность ResNet50 ломает ансамбль |

При 32 train-картинках за фолд каждый дополнительный обучаемый слой увеличивает риск переобучения. Frozen ResNet18 даёт **меньше параметров и более стабильный результат**, что важнее единичного высокого score у более тяжёлой модели.

### 2.3. 5-Fold Stratified Cross-Validation

При 40 картинках один train/val split даёт val = 8 примеров, и accuracy на нём принимает только дискретные значения (0.5 / 0.625 / 0.75 / 0.875) — оценка качества модели становится шумной.

**Решение:** `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`.
- Каждая картинка попадает в val ровно один раз → честная **OOF**-оценка на всех 40 примерах.
- Усреднение softmax-вероятностей 5 моделей на test → ансамбль.

### 2.4. Test-Time Augmentation (TTA)

Для каждой test-картинки усредняются softmax-вероятности от 6 прогонов:
- 1 базовый (без аугментаций, только Resize+Normalize)
- 5 со случайными аугментациями (RandomCrop, Flips, RandomRotation(20°))

### 2.5. Гиперпараметры

| Параметр | Значение | Обоснование |
|---|---|---|
| `IMG_SIZE` | 224 | стандартный вход для ResNet, предобученной на ImageNet |
| `BATCH_SIZE` | 8 | 32 train / 8 = 4 батча на эпоху |
| `NUM_EPOCHS` | 25 | дольше нет смысла — тренируется только FC слой |
| `LR` | 1e-3 | стандарт Adam для transfer learning |
| `WEIGHT_DECAY` | 1e-4 | L2-регуляризация |
| `DROPOUT` | 0.3 | регуляризация на голове |
| Optimizer | Adam | устойчив, хорошо работает с фиксированным backbone |
| Scheduler | ReduceLROnPlateau(factor=0.5, patience=3) | снижает LR, когда val acc перестаёт расти |
| Loss | CrossEntropyLoss | стандарт для классификации |
| SEED | 42 | воспроизводимость |

## 3. Предобработка и аугментации

### 3.1. Train pipeline

```python
Resize((256, 256))
RandomCrop(224)
RandomHorizontalFlip()
RandomVerticalFlip()
RandomRotation(30)
ColorJitter(brightness=0.3, contrast=0.3, saturation=0.3, hue=0.05)
RandomPerspective(distortion_scale=0.2, p=0.3)
ToTensor()
Normalize(mean=ImageNet, std=ImageNet)
```

Обоснование выбора:
- `Flip` (H+V) — тарелка симметрична, ориентация не несёт информации.
- `Rotation(30°)` — съёмка с разных углов.
- `ColorJitter` — разное освещение и баланс белого.
- `RandomPerspective` — лёгкая 3D-деформация.
- `Normalize` под ImageNet статистики — обязательно для pretrained моделей.

### 3.2. Eval/Test pipeline

```python
Resize((224, 224)) → ToTensor() → Normalize(ImageNet)
```

Детерминированный пайплайн без случайностей — для воспроизводимости.

### 3.3. Что не сработало (отброшено)

В ходе экспериментов проверялись и были отброшены:
- `RandomErasing` — при 40 картинках разрушает важные текстурные признаки.
- `RandomGrayscale` — цвет даёт полезный сигнал, обесцвечивание его убирает.
- `RandomResizedCrop(scale=0.7..1.0)` — слишком агрессивно для тарелок, занимающих не весь кадр.
- `Label smoothing 0.1` + `AdamW` + `CosineAnnealingLR` — в сумме ухудшили public LB с 0.83 до 0.72.

## 5. Обучение и результаты

### 5.1. Метрики 5-fold

| Fold | val acc |
|---|---|
| 1 | 0.750 |
| 2 | 0.625 |
| 3 | 0.875 |
| 4 | 0.500 |
| 5 | 0.625 |
| **mean ± std** | **0.675 ± 0.127** |

### 5.2. OOF (Out-Of-Fold) на 40 train-картинках

| Метрика | Значение |
|---|---|
| Accuracy | **0.675** |
| Macro F1 | **0.675** |
| Errors | 13 / 40 |

**Classification report:**

```
              precision    recall  f1-score   support
   cleaned        0.67       0.70      0.68       20
     dirty        0.68       0.65      0.67       20
    accuracy                            0.68       40
   macro avg      0.68       0.68      0.67       40
```

### 5.3. Confusion matrix (OOF)

```
              Predicted
              cleaned  dirty
True cleaned     14      6
     dirty        7     13
```

Файл: `outputs/confusion_matrix.png`

Ошибки симметричны: 6 чистых → грязные, 7 грязных → чистые. Систематического перекоса в одну сторону нет.

### 5.4. Графики обучения

`outputs/training_history.png` — кривые loss и accuracy по эпохам для fold 1.
- Train loss падает, train acc растёт до ~0.85.
- Val метрики стагнируют — типично для очень маленького val-сета (8 картинок).
- Это объясняет, почему за основу оценки берётся OOF на 40 картинках и public LB на 744 картинках, а не кривые одного фолда.

### 5.5. Анализ ошибок

`outputs/validation_errors.png` — первые 8 из 13 неверно классифицированных train-картинок (всего 13 ошибок из 40).

**Типичные паттерны:**
- Чистые тарелки с тенями/бликами → классифицируются как грязные.
- Грязь, мало контрастная с цветом тарелки (светлая еда на белой тарелке) → пропускается.
- Прозрачные/стеклянные тарелки — модель не уверена, таких примеров мало в train.

### 5.6. Результат на kaggle public LB

| Submission | Public LB |
|---|---|
| ResNet18 5-fold + TTA (финальный) | **0.83467** |

OOF accuracy = 0.675 vs public LB = 0.835. Разрыв объясняется:
1. Test (744 картинки) в 18× больше, чем OOF (40) → метрика менее шумная.
2. TTA применяется только на test, не на OOF.
3. На test каждая картинка получает усреднение от всех 5 моделей; в OOF — только от той, которая её не видела (1 модель на пример).

## 6. Выводы и направления улучшения

### 6.1. Выводы

- **Transfer learning с замороженным backbone** работает на few-shot задачах с 40 картинками лучше, чем разморозка дополнительных блоков.
- **K-fold CV** обязателен, когда один val-split даёт < 20 картинок — иначе оценка качества модели нестабильна.
- **Стабильность важнее единичного score:** ResNet50 показал 0.853 на public LB при разбросе по фолдам std=0.146, ResNet18 — 0.835 при std=0.127.

### 6.2. Рекомендации для дальнейшего улучшения

1. **Сегментация тарелки от фона (GrabCut / U-Net)** + аугментация фона. Топ-решения kaggle используют этот приём, чтобы убрать сигнал от фона и оставить только саму тарелку.
2. **EfficientNet-B0/B1** — компромисс по числу параметров между ResNet18 и ResNet50.
3. **Self-supervised pretraining** на 744 test-картинках (SimCLR / DINO) перед классификацией — задействует unlabeled данные.
4. **Pseudo-labeling**: уверенные предсказания на test (p > 0.95) добавить в train и переобучить.
5. **MixUp / CutMix** — дополнительная регуляризация через смешивание примеров.
6. **Многосидовый ансамбль:** обучить несколько 5-fold с разными `random_state` и усреднить.
